## carto 点云压缩

设计目的:

```
降低内存消耗
```

设计思路:

```
以1mm*1mm*1mm形成栅格, 根绝栅格进行编号, 储存该编号, 达到点云压缩目的

思考: 
    实际上思路与序列化与反序列化思路类似, 知识这里储存的是point_index(int32)
    但一切内存的消耗均会导致搜索的运算消耗增加, 本质是以时间换空间的消耗
```

拓展:

```
这种思路同样可以使用在grid_map上用于降低内存消耗, 原来是[Index2d(x, y), value]  ->  [uint(flat_index), value]
```
